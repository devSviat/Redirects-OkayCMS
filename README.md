# Модуль Redirects для OkayCMS

Модуль додає зручне керування редіректами в адмінці: можна створювати 301/302 правила, імпортувати та експортувати їх через CSV, бачити статистику спрацювань і автоматично створювати редіректи при видаленні товарів.

## Можливості

### Керування редіректами в адмінці

- Додавання та редагування редіректів.
- Підтримка кодів:
  - `301` (постійний)
  - `302` (тимчасовий)
- Увімкнення/вимкнення кожного правила.
- Пошук і фільтри по списку редіректів.

### Типи правил

- **Точний (`exact`)** — для одного конкретного URL.
- **Шаблон (`pattern`)** — для групи URL.
- У шаблоні можна використовувати змінну `$slug`.
- Якщо підходять і `exact`, і `pattern`, пріоритет має **`exact`**.

### Мультимовні редіректи

- Опція `is_lang` дозволяє застосовувати правило також до URL з мовним префіксом (`/en`, `/ua` тощо).

### Імпорт та експорт CSV

- Імпорт редіректів з CSV.
- Можна оновлювати існуючі редіректи за `from_url`.
- Експорт усіх редіректів у CSV.
- Є шаблон CSV для швидкого старту.

### Статистика

- Кількість спрацювань (`hits`).
- Дата останнього переходу (`last_hit_at`).

### Автоматичні редіректи при видаленні товару

- Після видалення товару модуль може автоматично створити редірект зі сторінки товару на його категорію.

## Вимоги

- OkayCMS v4.5.2

## Встановлення

### Варіант 1: Скачування з GitHub

1. Скачайте останню версію модуля з [останнього релізу на GitHub](https://github.com/devSviat/Redirects-OkayCMS/releases/latest).
2. Розпакуйте архів на сервері в каталог: `{OkayCMS_root}/Okay/Modules/`

### Варіант 2: Встановлення через Git

1. Виконайте команду в директорії `{OkayCMS_root}/Okay/Modules/Sviat`:
   ```bash
   git clone https://github.com/devSviat/Redirects-OkayCMS.git Redirects
   ```

### Результат встановлення

Шлях до модуля має бути таким:
```bash
{OkayCMS_root}/Okay/Modules/Sviat/Redirects/
```

Після встановлення перейдіть в адмін-панель → Модулі → Знайдіть «Redirects» → Натисніть «Встановити».

## Налаштування і використання

### Додавання редіректу

1. Перейдіть в адмін-панель → SEO → Редіректи.
2. Натисніть «Додати редірект».
3. Заповніть:
   - `URL звідки` (без домену, наприклад `old-page`)
   - `URL куди` (без домену, наприклад `catalog/new-page`)
   - тип (`exact` або `pattern`)
   - статус (`301` або `302`)
4. За потреби увімкніть мультимовність (`is_lang`).
5. Збережіть правило.

### Імпорт CSV

- Обов’язкові колонки: `from_url`, `to_url`.
- Додаткові: `name`, `status`, `enabled`, `is_lang`, `type`.
- Підтримуються `exact` і `pattern`.
- Для `pattern` можна використовувати `$slug`.

## Важливо

- Вказуйте URL **без домену** (відносні шляхи).
- Для постійного перенесення використовуйте `301`, для тимчасового — `302`.
